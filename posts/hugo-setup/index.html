Hugo Setup

2022-09-18

This is not supposed to be an in-depth guide on all things Hugo. I rather intend to note down a few key essentials for me to come back to later.

Adding new posts

New posts can be added by running

$ hugo new posts/postname.md

in the root directory of the blog. The article will be automatically listed under posts on the page. If you plan on adding a separat page, that isnt supposed to be a post, you could add it under the page subdirectory. To reach it you then have to specify a permalink with the slug variable, because a page created like this wont be indexed on the visible blog.

Deploying the updates to the actual blog

After adding / editing you most likely want to publish the changes and deploy them. To do this, you first of all have to “compile” the static files from the hugo project by running

$ hugo -t themename

in the root directory of the project. This creates the static site files in the public directory. These now have to be pushed to the server. I set up a git submodule with the actual blog files in this directory by following this guide. After running

$ cd public
$ git add .
$ git commit -m "Your commit message"
$ git push origin main

the changes will appear on the actual blog shortly after.
